Написать REST API приложение для ведения учёта книг. Должны присутствовать следующие сущности: автор, книга, издательство.
Данные передавать в формате json. Создать конечные точки (endpoints) для чтения, создания, удаления и изменения каждой сущности.
Сущности хранить в БД. Детализацию описания сущности выбрать самостоятельно.
Создать конечные точки для извлечения следующей информации:
1) Все книги заданного издателя (издателя задавать по первичному ключу БД).
2) Все книги заданного автора (автора задавать по первичному ключу БД).

Применять следующий стек:
Spring boot, Spring MVC(REST), Spring JPA(Hibernate), H2 database, Jackson.
Желательно, но не обязательно развернуть приложение в докере
